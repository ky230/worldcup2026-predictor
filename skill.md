## 二、数据与舆情预测决策环路 (Prediction Loop Workflow)

> **说明**：每次生成新比赛日 HTML 时，**按照以下三个阶段顺序执行**，不可跳过或合并阶段。Loop 目标是把专业数据与场外情报全部榨干后，再输出 HTML 面板。详细的数据源链接、搜索Query及标注规范，请统一参阅 [data_sources.md](file:///Users/ky230/Desktop/FF2026/reference/data_sources.md)。

### 【阶段一】专业数据深挖 · 锚定基础实力

**执行原则**：不只看阵容本身，重点分析「形势层」。对每场比赛，依次查阅以下具体数据页面与指标：

1. **FotMob 核心指标与页面查询**：对对阵双方在数据手册中查找具体的 FotMob URL，**首选且唯一的抓取手段是运行仓库内置脚本，严禁使用慢速的浏览器子代理**：
   - **单场详细数据抓取**：运行 [fetch_match_details.py](file:///Users/ky230/Desktop/FF2026/scripts/fetch_match_details.py) 脚本：`python3 scripts/fetch_match_details.py <match_url> <date_dir>/json`。脚本会自动请求网页，解析 HTML 中的 `<script id="__NEXT_DATA__">`，并将包含首发名单与 xy 战术板坐标、裁判、实时天气、近期状态（Form）、缺阵伤停、Insights 事实和 H2H 等信息的 JSON 写入对应的 `date/json` 文件夹。
   - **小组实时排名抓取**：运行 [fetch_league_table.py](file:///Users/ky230/Desktop/FF2026/scripts/fetch_league_table.py) 脚本：`python3 scripts/fetch_league_table.py <date_dir>/json`。脚本会自动抓取世界杯最新的全小组积分榜及“成绩最好的小组第三”数据，写入 `<date_dir>/json/standings.json`。**必须根据小组实时排名（如队伍是否急需拿分出线、是立足打防反还是大比分血拼）以及小组赛轮次，在 AI 预测报告中定性研判各队的战意与血拼程度**。
   - **生成的 HTML 面板中的首发阵型、球员姓名与位置，必须与 FotMob JSON 数据完全保持一致，做到“JSON 坐标所见即面板所见”，绝对不允许出现位置错位或颠倒**。
   - **球员多维数据**：打开 [data_sources.md](file:///Users/ky230/Desktop/FF2026/reference/data_sources.md) 指引的 [球员统计数据](https://www.fotmob.com/leagues/77/stats/world-cup/players) 及其子项直达链接（包含进球 Goals、助攻 Assists、期望进球 xG、期望助攻 xA、创造重大机会 Big chances created、黄牌 Yellow Cards 与 FotMob 评分），核对两队核心球员的个人数据。对于黄牌情况，直接访问 [黄牌统计数据](https://data.fotmob.com/stats/77/season/24254/yellow_card.json) 接口以确认各队已吃牌的球员名单，并在战术板上为这些球员的名字后缀添加 `🟨`。关于红牌或累计黄牌停赛、伤病停赛等情况，首发阵容 Preview 中会自动将他们移出预计首发并放入 `unavailable` 中，因此无需担心。
   - **团队攻防战术**：打开 [data_sources.md](file:///Users/ky230/Desktop/FF2026/reference/data_sources.md) 指引的 [球队统计数据](https://www.fotmob.com/leagues/77/stats/world-cup/teams)。核对两队的 Expected Goals (xG) 与 Expected Goals Conceded (xGA)（即 Defense 分类下的 xG conceded）；查看 Average possession 和 Accurate passes per match 判定谁掌控中场；使用 **Possession won final 3rd per match**（在 Defense 分类中）作为逼抢强度的直接代理指标（因 FotMob 无原生 PPDA）。

2. **裁判与比赛环境核对**：直接从上述脚本解析得到的 **FotMob 单场 Match Details 数据**（`props.pageProps.content.matchFacts.infoBox` 与 `content.weather` 模块）中查阅核对：
   - **主裁判信息**：提取裁判指派安排，必要时检索其历史执法尺度与给牌率，判定防守型/侵略性战术修正；
   - **球场与环境**：确认比赛球场、城市、实时气温（必须包含在 HTML 信息条中）与降雨等。如果为高原城市（如墨西哥城、瓜达拉哈拉等 1500m 以上海拔），或实时气温 ≥ 32°C，必须触发本手册【3-A】中的胜率修正规则。

### 【阶段二】场外雷达 · 更衣室舆情侦察

**执行方针**：场外情报是**概率校正因子**。场外情报的价值在于识别「纸面实力与实际发挥之间的折扣」。
1. **分级舆情与伤停核对**：根据 [data_sources.md](file:///Users/ky230/Desktop/FF2026/reference/data_sources.md)【第三部分 3-A】所确立的分级优先级检索舆情和实时伤停：
   - **第一优先级**：优先查阅 **FotMob 单场比赛详情页**（Insights & News 实时伤停/红黄牌更新、Team Form 走势、About the Match 赛事信息）。
   - **第二优先级**：查阅 **Sofascore 新闻专区** 及 **FIFA 官方新闻** 验证世界杯赛事大盘新闻与伤退官宣。
   - **第三优先级**：使用 [data_sources.md](file:///Users/ky230/Desktop/FF2026/reference/data_sources.md)【3-B】所列的 **Google 搜索 Query 模板** 或官方社媒，核查深层舆情（如更衣室内讧细节、战术泄密等）。
2. **多信源校验**：依据 [data_sources.md](file:///Users/ky230/Desktop/FF2026/reference/data_sources.md) 的【3-C 舆情爆料源信源分级与可信度目录】核查新闻可信度。只有当 1 个以上一级媒体，或 2 个以上二级媒体独立证实后，方可启动胜率下调机制。
3. **后勤与环境排查**：直接在 FotMob 单场比赛详情页（About the Match 模块等）检索开球时间，核查当地海拔和气温是否触发本手册【3-A】节的概率扣分规则（无需考虑主客场差旅时区差）。

### 【阶段三】信息融汇与研判决策逻辑

**执行原则**：「用数据定锚，用情报调偏」。先确定基础概率（来自阶段一的硬数据），再用阶段二的情报作为「修正因子」。禁止逆序操作。

#### 3-A. 基础概率锚定公式

```
基础胜率 = 
  近期状态(40%) × 形态分
+ 硬实力底盘(30%) × 世界档位
+ 历史交锋(15%) × 中立场胜率 (🚨 历史交锋限制：仅参考 5 年之内的交手记录；年份超过 5 年的交手记录无参考价值，该项 15% 权重直接归 0 均摊至近期状态 47% 与硬实力 38% 中)
+ 情境因素(15%) × [东道主加成 / 天气/海拔修正 / 淘汰赛抗压值]
```

情境因素中**天气/海拔修正标准值**：
* 气温 ≥ 32°C：高位逼抢体系（PPDA<8）欧洲队，**胜率 -3%~-5%**；低重心防守反击体系，**胜率 +2%~+3%**（受益方）。
* 海拔 ≥ 1500m（如丹佛、墨西哥城）：非本洲球队，**胜率 -4%~-8%**；本洲球队 +4%~+6%。
* 前一天深夜赛后短暂休息（<48h）：任何球队，**体能系数 -3%**。

#### 3-B. 情报修正因子（场外减分/加分系数）

| 情报类型 | 确认门槛 | 概率修正 | 波胆影响 |
|---------|---------|---------|---------|
| **更衣室内讧（确认）** | ≥2个独立一级媒体源 | **热门方 -8%~-15%** | 意外比分权重+5%，放大冷门尾部概率 |
| **更衣室内讧（单源爆料）** | 仅1个媒体源 | **-3%~-5%** | 在舆情哨中标注，波胆不变 |
| **核心球员伤停（官方确认）** | FotMob移出首发 OR 官方声明 | 按球员影响力 **-5%~-20%** | 主力核心级（如梅西/姆巴佩）≥-15% |
| **本届大赛中球队已露出疲软信号** | 首轮实际比赛结果 | 下修信心系数，胜率区间扩散 | 防冷概率+3% |

#### 3-C. 国家/球队内讧体质校准表
以下为历史大赛数据校准的"内讧折扣系数"。当爆料发生时，按此表决定修正力度而非一刀切。
* 🇫🇷 **法国** (内讧体质：🔴 HIGH)：历史多发更衣室事件（2010罢训）。确认内讧 → **-15%**。
* 🇨🇲 **喀麦隆** / 🇳🇬 **尼日利亚** (内讧体质：🔴 HIGH)：常有主帅/球员冲突或权力争夺。确认内讧 → **-15%~-20%**。
* 🇧🇪 **比利时** (内讧体质：🟠 MED-HIGH)：荷语/法语圈派系分化背景音强。确认内讧 → **-10%~-12%**。
* 🇵🇹 **葡萄牙** (内讧体质：🟠 MED)：C罗话语权与球星定位风波。C罗相关更衣室爆料 → **-8%**。
* 🇦🇷 **阿根廷** (内讧体质：🟡 MED)：梅西体系稳定时较低（-3%），若核心有争议则下调 **-10%**。
* 🇳🇱 **荷兰** (内讧体质：🟡 LOW)：历史多内讧但上场独立性强。确认内讧 → **-2%~-3%**。
* 🇩🇪 **德国** / 🏴󠁧󠁢󠁥󠁮󠁧󠁿 **英格兰** (内讧体质：🟡 LOW)：管控较严。确认内讧 → **-5%**。
* 🇯🇵 **日本** / 🇰🇷 **韩国** (内讧体质：🟢 VERY LOW)：集体荣誉文化，基本不修正，仅在舆情哨注明。

#### 3-D. 研判决策流程与平博量化计算

```
Step 1: 确认「热数据」 → 阶段一的 xG、近5场评分、伤停、天气。
Step 2: 确认「冷情报」 → 阶段二的内讧/伤停烟雾弹分级。
Step 3: 依据硬数据锚定基础概率（Step 1）。
Step 4: 叠加情报修正因子进行概率微调，得出最终 AI 预测胜率（$P_H, P_D, P_A$）（平局概率合理，单队胜率 ≤ 85%）。
Step 5: 爬取平博（Pinnacle）实时赔率。由于平博具有强烈的 Cloudflare 防御，必须使用浏览器子代理（Browser Subagent）访问：
   - 世界杯主页：https://www.pinnacle.com/zh-CN/soccer/fifa-world-cup/matchups/#all 找到当日赛程。
   - 点击具体比赛详情页（例如：https://www.pinnacle.com/zh-CN/soccer/fifa-world-cup/argentina-vs-austria/1620858180/#all）。
   - 在详情页中定位并抓取以下盘口的水位：
     1) 1X2（胜平负）赔率 -> 对应折叠板中的「输赢盘 - 比赛」；
     2) 大小球（2.5）赔率 -> 对应折叠板中的「总分盘 - 比赛」（提取大盘2.5与小盘2.5）；
     3) 正确得分（正确比分）赔率 -> 对应折叠板中的「正确得分」（若折叠，需点击展开，提取所有标准比分的实时水位）。
   - 将爬取的数据统一保存为当天文件夹下的 `json/pinnacle_odds.json`。
Step 6: 计算全场进球期望与泊松拟合：
   - 提取大小球小盘 2.5 赔率 $O_U$ 与大盘 2.5 赔率 $O_O$，通过归一化去除博彩抽水，计算小盘 2.5 无偏概率 $P_{\text{Under}}$.
   - 求解泊松累积概率方程：$e^{-\lambda_{\text{total}}} (1 + \lambda_{\text{total}} + \frac{\lambda_{\text{total}}^2}{2}) = P_{\text{Under}}$。通过二分法求解全场期望进球数 $\lambda_{\text{total}}$。
   - 在 $\lambda_H + \lambda_A = \lambda_{\text{total}}$ 的约束下，通过 1D 网格搜索迭代寻找使 Poisson 分布导出的主胜、平局、客胜概率与前文 AI 预测胜率（$P_H, P_D, P_A$）均方误差最小的 $\lambda_H, \lambda_A$，从而计算出完整的比分概率矩阵。
Step 7: 计算投注期望值与资金比重（半凯利模型）：
   - 对 1X2 和让球盘口计算数学期望值 $EV = P_{\text{win}} \cdot b - P_{\text{lose}}$，其中 $b = \text{Odds} - 1$。
   - 让球盘口若存在平盘走水（Push），则使用修正公式：$f^* = \frac{p_{\text{win}} \cdot b - p_{\text{lose}}}{b \cdot (1 - p_{\text{push}})}$。对于半/一等拆分盘口，使用 1D 期望对数效用最大化算法求解满额凯利比例 $f^*$。
   - 使用半凯利公式（Half-Kelly, $k = 0.5$），以 200 元总本金为基准，计算各项正 EV 玩法的下注金额。
Step 8: 量化输出比分建议（3 稳 + 3 冷）：
   - 3个稳波胆比分 (Steady Scores)：泊松概率最高的前 3 个比分，附带平博真实赔率。
   - 3个冷门比分 (Upset/Surprise Scores)：根据强弱态势在排除稳胆比分后，从以下逻辑冷门中筛选出泊松概率最高的 3 个比分并匹配平博真实赔率：
     1) 强队为 Home 时（AI胜率 > 55%）：防冷平局（0-0, 1-1, 2-2）或下盘队偷球险胜（0-1, 1-2）；
     2) 强队为 Away 时（AI胜率 > 55%）：防冷平局（0-0, 1-1, 2-2）或下盘队偷球险胜（1-0, 2-1）；
     3) 实力接近的对局（胜率均 < 55%）：平局（2-2, 3-3）或任一方零封小胜（1-0, 0-1）。
Step 9: 将 1X2 和让球的期望值、半凯利投注金及 3稳 + 3冷比分与赔率信息，全部编译并集成在每日赛事卡片面板中。
Step 10: 在「更衣室晴雨表」模块写入当前侦察结论。
```

## 四、球队资料库（核心依据）

### 夺冠热门档

**阿根廷**（J组）：梅西（39岁，迈阿密国际，最后一舞）。**战术特色**：**麦卡利斯特与恩佐双核调度默契**，德保罗在中右路护航，劳塔罗（高位拦截为梅西省体能）与阿尔瓦雷斯在单双锋线切换。**隐忧与雷区**：梅西体能不足以支撑全场逼抢；奥塔门迪身后存在反击打穿的真实风险。德保罗停赛时防守强度下降 15%-20%。

**西班牙**（H组）：**战术特色**：**亚马尔（18岁）与尼科·威廉姆斯两翼齐飞**，中场罗德里（定海神针）覆盖面大，佩德里或奥尔莫担任前插攻击手。**隐忧与雷区**：年轻主力大赛逆境经验偏少；亚马尔未经历深度打压考验。中锋莫拉塔心理脆弱，罗德里单防跑动时两侧身后易暴露。
